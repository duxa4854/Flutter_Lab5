## Что изучили
В ходе выполнения лабораторной работы были освоены следующие концепции:
Асинхронное программирование в Dart: Понимание принципов работы Future<T>, ключевых слов async и await для неблокирующего выполнения операций.
Работа с сетью: Использование пакета http для отправки GET-запросов к внешним API и парсинг JSON-ответов с помощью dart:convert.
Управление состоянием (State Management): Практическое применение setState() в StatefulWidget для обновления UI при изменении данных (статус загрузки, получение URL изображения, обработка ошибок).
Обработка ошибок: Использование конструкции try/catch для корректной обработки сетевых ошибок и отсутствия интернета.
Работа с изображениями: Различия между загрузкой изображений из сети (Image.network) и из активов проекта (Image.asset).
Ответы на контрольные вопросы
1. Что такое Future<T>? Чем отличается от обычного возвращаемого значения?
Future<T> — это объект, представляющий собой «обещание» получить значение типа T или ошибку в будущем.
Отличие: Обычное возвращаемое значение доступно сразу после выполнения функции синхронно. Future же означает, что результат будет получен асинхронно (например, после запроса к серверу), и код должен явно ожидать его завершения или подписаться на результат.
2. Что делает await? Блокирует ли он весь поток выполнения?
Ключевое слово await приостанавливает выполнение текущей асинхронной функции до тех пор, пока связанный с ним Future не завершится (не вернет результат или ошибку).
Блокировка потока: Нет, await не блокирует основной поток выполнения (UI thread). Пока функция ожидает ответа, Flutter может продолжать обрабатывать другие события, перерисовывать интерфейс и реагировать на действия пользователя. Это обеспечивает отзывчивость приложения.
3. Зачем setState() вызывается дважды в _fetchPhoto()?
Вызов setState() необходим для уведомления Flutter об изменении состояния виджета и запуска перестройки интерфейса (build):
Первый вызов (в начале): Устанавливает _isLoading = true и сбрасывает предыдущие данные (_imageUrl = null, _errorMessage = null). Это нужно, чтобы немедленно показать индикатор загрузки (CircularProgressIndicator) и скрыть старое фото или ошибку.
Второй вызов (в конце): Устанавливает _isLoading = false и сохраняет полученный URL (_imageUrl) или текст ошибки (_errorMessage). Это нужно, чтобы скрыть индикатор загрузки и отобразить новое изображение или сообщение об ошибке.
4. Почему кнопке передаётся _fetchPhoto без скобок, а не _fetchPhoto()?
_fetchPhoto (без скобок): Передается ссылка на функцию (callback). Функция не выполняется в момент построения виджета. Она будет вызвана только тогда, когда пользователь действительно нажмет на кнопку (событие onPressed).
_fetchPhoto() (со скобками): Это вызов функции. Если написать так, функция выполнится сразу же при каждом перестроении виджета (при каждом build), что приведет к бесконечному циклу запросов к серверу и ошибкам, так как onPressed ожидает функцию типа void Function()?, а не результат выполнения void.
